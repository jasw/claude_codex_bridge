Expected default result:

- All normal tests pass.
- `test/test_provider_memory_external_context.py` is skipped unless explicitly
  enabled with `CCB_REAL_PROJECT_MEMORY_CHECK=1`.

Coverage:

- Provider source ownership policy for Claude, Codex, OpenCode, Gemini, and
  unknown providers.
- Claude route-mode install/uninstall cleanup preserves unmarked user
  `~/.claude/rules/ccb-config.md` files and removes CCB-marked legacy external
  config files.
- Seed-aware `.ccb/ccb_memory.md` upgrade for unedited old generated templates,
  with edited old files preserved. This includes seed-metadata-backed upgrades
  and exact known generated legacy-template matches when seed metadata is
  missing.
- Provider-user-memory install-residue filtering for marker pairs and legacy
  collaboration-rule sections.
- Filter conservatism: isolated markers, unrelated user text, and user
  paragraph spacing are preserved.
- Claude generated `.claude/CLAUDE.md` includes filtered provider user memory,
  shared CCB memory, and agent private memory, but not project `CLAUDE.md`.
- Codex generated `AGENTS.md` includes filtered provider user memory, shared
  CCB memory, and agent private memory, but not project `AGENTS.md`.
- OpenCode generated runtime memory bridge includes shared CCB memory and agent
  private memory, but not project `AGENTS.md`; generated config still preserves
  project `opencode.json` instructions and appends the CCB bridge instruction.
- Gemini remains unchanged and still includes project `GEMINI.md` pending its
  native loading audit.
